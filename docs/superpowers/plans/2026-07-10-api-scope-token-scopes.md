# API-SCOPE — Per-token read/write scopes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give each `/api/v1` personal access token a `read` or `read-write` scope, enforced centrally so read-only tokens are rejected on any write.

**Architecture:** Add a `scope` column to `api_token`. The token resolver returns the scope; the single shared route wrapper `withUser()` rejects a `read` token on any non-`GET` method with a 403. Minting gains a scope selector; existing tokens backfill to `read-write` so nothing breaks.

**Tech Stack:** Next.js (App Router) route handlers, Drizzle ORM + drizzle-kit migrations, Postgres, TypeScript, React client component for settings.

## Global Constraints

- **No test runner exists in this repo.** Verification is `npm run typecheck` + `npm run lint` (+ `npm run build` at the end) plus a live curl smoke against the dev server. Do NOT add jest/vitest — follow the existing convention.
- **All commands run from `app/`** (the Next.js project root): `cd app` first.
- **Dev server port is 3210** (`next dev -p 3210`).
- **Token plaintext is shown once**; only the SHA-256 hash is stored. Never log or persist plaintext.
- **Scope values are exactly** `'read'` and `'read-write'`. No other values. No granular per-resource scopes.
- **New tokens default to `'read-write'`;** existing rows backfill to `'read-write'` via the column default.
- **Scope is immutable per token** — no in-place edit; revoke + re-mint to change.

Update `resolveApiToken` to select and return `scope`:

```ts
export async function resolveApiToken(
  plaintext: string,
): Promise<{ userId: string; scope: ApiTokenScope } | null> {
  if (!plaintext.startsWith(TOKEN_PREFIX)) return null;
  const rows = await db
    .select({
      id: apiTokens.id,
      userId: apiTokens.userId,
      scope: apiTokens.scope,
    })
    .from(apiTokens)
    .where(
      and(
        eq(apiTokens.tokenHash, hashToken(plaintext)),
        isNull(apiTokens.revokedAt),
      ),
    )
    .limit(1);
  const row = rows[0];
  if (!row) return null;

  await db
    .update(apiTokens)
    .set({ lastUsedAt: new Date() })
    .where(eq(apiTokens.id, row.id));

  return { userId: row.userId, scope: row.scope };
}
```

- [ ] **Step 2: Add the enforcement check**

```ts
export async function withUser(
  req: Request,
  handler: (userId: string) => Promise<NextResponse>,
): Promise<NextResponse> {
  try {
    const { userId, scope } = await requireApiUser(req);
    if (scope === 'read' && req.method !== 'GET') {
      throw new ServiceError('forbidden', 'Token lacks write scope');
    }
    return await handler(userId);
  } catch (err) {
    return apiErrorFrom(err);
  }
}
```

(`ServiceError` is already imported in this file; `forbidden` already maps to 403 in `api-response.ts`.)
